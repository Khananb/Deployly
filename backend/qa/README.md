# Deployly Automated QA Framework

This is the automated QA runner for Deployly. It tests the end-to-end functionality of Authentication, Website Creation, ZIP Uploads, Deployments (Static & Node.js), Redeployments, and Deletions.

## Prerequisites
- Node.js 18+ (uses native `fetch`)
- The backend server must be running (either locally or on a VPS).

## How to Run

By default, the tests run against `http://localhost:3000/api`.
```bash
node backend/qa/qa_runner.js
```

### Run against Production VPS
To test the production API, pass the `QA_BASE_URL` environment variable:

**Windows (PowerShell):**
```powershell
$env:QA_BASE_URL="https://deployly.online/api"; node backend/qa/qa_runner.js
```

**Linux/Mac:**
```bash
QA_BASE_URL=https://deployly.online/api node backend/qa/qa_runner.js
```

### Run a Specific Test
You can skip other tests and only run a specific suite using the `--only` flag. This matches against the test name or filename.
```bash
node backend/qa/qa_runner.js --only auth
node backend/qa/qa_runner.js --only static
node backend/qa/qa_runner.js --only redeploy
node backend/qa/qa_runner.js --only node
node backend/qa/qa_runner.js --only delete
```
*(Note: Some tests like 'redeploy' or 'delete' rely on state generated by earlier tests like 'static', so running them in isolation might skip parts of the execution).*

## Automated Reports
Every run generates a report string and appends it to:
`backend/qa/reports/YYYY-MM-DD.txt`

You can use these logs to compare history.

## Adding New Tests
1. Create a new `myfeature.test.js` file in `backend/qa/`.
2. Export an async `run(state)` function and a `name`.
3. Add the filename to the `TEST_FILES` array in `backend/qa/qa_runner.js`.
